Platform Agent Pipeline

Architect → Implementer → Code Reviewer
Multi-pass, cross-model verified, auto-fixing

How It Works

Every feature goes through three specialized agents, each independently verified by a second AI model. No code ships without at least two full review cycles.

Stage 1
Architect
Understands the requirement, researches existing code, assesses scope, and produces a detailed design document before any code is written.
Explore subagent researches codebase
Write architecture document
GPT-5.4 scrutinizes for gaps
Revise based on scrutiny findings
Handoff
Stage 2
Implementer
Follows the architecture document task by task — writing code, tests, and migrations. Self-verifies before requesting review.
Cycle 1 — Code Reviewer runs full review
Fix all — every finding auto-fixed
Cycle 2 — re-review fixed code from scratch
Assess — clean or fix remaining
Invokes
Stage 3
Code Reviewer
Reviews every changed file across 10 categories. Findings are cross-verified by a second AI model, then returned for auto-fix.
Review all files across 10 categories
GPT round 1 — independently verify findings
Reconcile — accept, reject, adjust
GPT round 2 — verify merged review
Return — findings sent back for auto-fix
3 primary agents, 3 GPT subagents, 5 cross-model verification passes.
Architect designs it. Implementer builds it. Reviewer validates it. All findings are auto-fixed and re-verified.

Research Pipeline

Standalone agent for producing verified documentation. Not connected to the implementation pipeline — invoked independently when research or documentation is needed.

Standalone
Researcher
Researches a topic using codebase, web, and file tools. Drafts a comprehensive .md document, then gets it proofread and fact-checked by two independent GPT subagents.
Research — gather sources, explore codebase
Draft — write structured .md document
Self-proofread — fix grammar, clarity, flow
GPT Proofreader — grammar, structure, consistency
GPT Accuracy Checker — facts, logic, sources
Verify & apply — dedupe, accept or reject findings

1
Research & Draft
Thoroughly research using search, web/fetch, read. Write comprehensive .md with proper structure. Save to documents/research/.
2
Self-Proofread
Re-read draft critically. Fix grammar, clarity, structural issues. Ensure completeness and flow.
3
GPT Proofread (Cross-Model)
Send document to GPT-5.4 proofreader for independent review.
↳ GPT Research Proofreader GPT-5.4 · Free
Checks 5 categories: grammar & spelling, clarity & readability, structure & flow, consistency, and completeness. Returns quoted issues with suggested fixes.
☠️ Context dies — findings returned
4
GPT Accuracy Check (Cross-Model)
Send document to GPT-5.4 accuracy checker for independent verification.
↳ GPT Research Accuracy Checker GPT-5.4 · Free
Checks 5 categories: factual accuracy, technical correctness, logical consistency, source validity, and currency. Returns verdicts: Accurate / Inaccurate / Uncertain / Outdated.
☠️ Context dies — findings returned
5
Verify & Apply
Deduplicate findings from both subagents. For each: verify independently, apply if correct, reject with reasoning if incorrect, research further if uncertain. Save final version.
6
Summary
Report: what was researched, file location, findings count from each subagent, accepted vs rejected with reasons.
